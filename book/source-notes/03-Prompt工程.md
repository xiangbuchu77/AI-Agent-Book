# 📘 第3章：Prompt工程（如何控制AI）

---

# 🎯 本章目标

学完本章，你将理解：

- 什么是Prompt
- 为什么Prompt能控制AI输出
- System / User Prompt区别
- Few-shot是什么
- 如何让AI稳定输出

---

# 🧠 1. 什么是Prompt？

一句话：

> Prompt = 给AI的指令

---

## 📌 举例

你对AI说：

帮我写一篇文章

这句话就是Prompt

---

# 🧠 2. 为什么Prompt重要？

因为：

AI不会主动思考目标，它只会响应输入

---

## 📌 类比

AI像一个超级实习生：

- 你怎么说
- 他就怎么做

---

# ⚙️ 3. Prompt的结构

Prompt一般包含：

- 角色（Role）
- 任务（Task）
- 限制（Constraint）
- 输出格式（Format）

---

## 📌 示例

你是一个资深AI工程师
请解释Transformer
要求：简单易懂
输出：分点结构

---

# 🧠 4. System vs User Prompt

| 类型 | 作用 |
|------|------|
| System | 定义AI身份 |
| User | 用户输入 |

---

## 📌 举例

System：

你是一个AI老师

User：

解释Transformer

---

# 🔥 5. Few-shot（示例学习）

给AI看例子：

输入：苹果 -> 输出：水果
输入：汽车 -> 输出：交通工具

AI会学习模式

---

# 🧠 6. CoT（思维链）

让AI一步一步思考：

请一步一步解释

效果：
- 更准确
- 更稳定

---

# ⚙️ 7. Prompt工程核心技巧

- 明确角色
- 明确任务
- 限制输出
- 提供例子

---

# 📊 8. Prompt结构模板

你是{角色}

任务：{任务}

要求：
- {限制1}
- {限制2}

输出格式：
{格式}

---

# 🎯 9. 面试常问

什么是Prompt？
> 控制AI输出的输入指令

Few-shot是什么？
> 用例子教AI

为什么Prompt重要？
> 因为AI完全依赖输入

---

# 📌 本章总结

- Prompt = 控制AI的方式
- 结构越清晰，输出越稳定
- Prompt就是指挥AI的语言
